About the Project

Travel Planner is a full-stack web application developed to simplify the process of planning and managing travel experiences. The platform allows users to explore destinations, manage personalized content, and securely interact with the system through authenticated access. The project follows modern backend and security best practices and is designed as a production-ready application.

Current Development Stage

At the current stage (Final Project), the application has evolved from earlier assignments into a complete and secure web system. The following core features have been successfully implemented:

  • Modular Backend Architecture: Built with Node.js and Express.js using a clear separation of routes, models, middleware, and configuration files.
  • Database Integration: MongoDB is used as the primary database with realistic domain-based data for destinations, including relationships and ownership control.
  • Authentication & Security: User authentication is implemented using sessions and bcrypt password hashing, ensuring secure login and logout functionality.
  • Authorization & Roles: Role-based access control (RBAC) is implemented with two roles: user and admin. Users can modify only their own data, while admins have extended management privileges.
  • Protected API Endpoints: All write operations are secured using middleware to prevent unauthorized access and enforce ownership rules.
  • Interactive Frontend: A responsive user interface that dynamically loads data from the backend API, supports filtering, favorites, and client-side persistence.

Future Improvements

Future development plans include advanced search and filtering, budget calculation tools, real-time data integration (such as weather APIs), and further UI enhancements to improve scalability and user experience.
